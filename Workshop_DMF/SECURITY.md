# Security Policy

## Supported Versions

Only the latest release receives security updates.

| Version | Supported          |
|---------|--------------------|
| latest  | :white_check_mark: |
| < latest| :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, use [GitHub Private Security Advisories](https://github.com/alisonpezzott/default-template/security/advisories/new)
to report a vulnerability privately.

When reporting, please include:

1. A description of the vulnerability and its potential impact.
2. Steps to reproduce the issue.
3. Any proof-of-concept code or screenshots (if applicable).
4. Your suggested fix, if you have one.

## Response Timeline

| Stage              | Target time |
|--------------------|-------------|
| Initial response   | 48 hours    |
| Confirmation       | 5 days      |
| Fix & disclosure   | 90 days     |

We will credit reporters who responsibly disclose vulnerabilities unless they request to remain anonymous.
